HTML Fundamentals and the World Wide Web

Still very simple, but I added sections for the Blog, Essays and Portfolio sections. For now, I’ve decided to create these sections as templates for when I actually begin filling them with content. Currently, all my sections are on the homepage. I will change this as soon as I develop my folder structure. My goal for now, is to make it website quite simple, first learning the basics of HTML and folder structures before moving onto move complex stuff. At the moment, I’m considering a folder structure similar to this:

  • Index.html
  • Blog Folder:
  • Blog1.html
  • Blog2.html
  • Blog3.html
  • Blog4.html
  • Portfolio Folder: Portfoliowork1.html
    • Portfoliowork2.html Portfoliowork3.html
  • Essay Folder: Essay1.html
    • Essay2.html Essay3.html
  • Images Folder: Image1.html
    • Image2.html

An article by Stony Brook University mentioned that a folder structure should be “pages should be organized logically into folders according to the sitemap. Each content area of the site must have a corresponding section folder. A section folder should always have a home page, also called an index page.” (Stony Brook University, 2024)

Hanli mentioned that adding alt text to images can be important in situations where the user is visually impaired or has a slow connection. This way the image can still be described to them. I will make sure to add alt text to my images.

Reflection on Reading

With regards to the reading by Moulthrop, I would like to focus on this particular quote “Hypertext is all about connection, linkage, and affiliation. Formally speaking, its universe is the one Thomas Pynchon had in mind when he defined “paranoia” as ‘the realization that everything is connected, everything in the Creation—not yet blindingly one, but at least connected. . . .'” (Moulthrop, 2003). This highlights what hypertext is all about, connection. Since websites use HTML (Hypertext Markup Language) to display information to users, it is important to note that everything on the website needs to be connected in some way. Whether it be code referencing another line of code from a different folder, or a webpage that links to another webpage, the experience needs to be almost seamless, with the only friction being the user’s internet connection. This is even more relevant today where there a so many websites that offer the same service or content, that any interruption for the user could result in them visiting another website instead, potentially losing a customer or viewer.

References

Moulthrop, S. 2003, 1991. You Say You Want A Revolution? Hypertext And The Laws Of Media, Post Modern Culture.
